# VideoRecover 平衡工作台设计

**日期：** 2026-07-17
**状态：** 已确认
**范围：** Web 管理台信息架构与视觉精简；后端 API、下载、转写、MCP 和数据模型不变。

## 目标

把当前同时强调巨型新建区、归档索引、状态横幅、视频资料、文案和导出网格的页面，收敛为一个紧凑、清晰的日常工作台。新建解析和历史任务保持同等重要；视频、发布原文、语音文案以及六种导出文件仍默认可见。

## 已确认的设计取舍

- 采用「平衡工作台」：顶部紧凑解析栏，下方左侧最近任务、右侧当前任务详情。
- 保留米色纸张、档案红、中文衬线标题的识别度。
- 减少英文装饰标签、粗重分隔线、超大标题、重复状态和无功能留白。
- 视频、原文、语音文案及 MP4、TXT 原文、TXT 语音、SRT、Markdown、JSON 全部保留并直接展示。
- Cookie 凭据移入顶部「设置」折叠面板；删除和重试移入详情「更多」菜单。

## 页面结构

### 1. 紧凑页头

页头高度控制在一个普通工具栏范围内，包含：

- VideoRecover 品牌名和小型「录」印章。
- 本地服务与 MLX Worker 状态。
- 「设置」入口及 Cookie 配置状态。

不再显示重复英文副标题。Cookie 表单仍使用密码输入框，仍加密保存到本机。

### 2. 解析栏

解析栏在桌面端为一行：小型标题、链接输入、语音提取选项、主按钮。提交中、成功或错误消息在输入栏下方就近显示，保留 `aria-live` 状态反馈。

解析栏不再使用首屏巨型宣传标题，也不占用大量纵向空间。

### 3. 平衡工作区

桌面端主工作区分两栏：

- 左栏约 14–17rem：最近任务、任务数量、状态和日期。
- 右栏占剩余空间：选中任务详情。

任务项不再显示装饰性序号；选中态同时使用背景、左侧强调线和 `aria-current`，不依赖颜色单独表达。

### 4. 任务详情

详情标题区包含：

- 任务状态、进度和消息。
- 视频标题、发布者。
- 「更多」菜单，其中包含按状态显示的重新处理和删除操作。

详情内容在宽屏采用视频约 30%、文案约 70% 的两列布局：

- 左侧：竖屏视频与四项紧凑元数据。
- 右侧：发布原文、语音文案和复制按钮。
- 底部：单行可换行的六个导出链接。

详情区不再重复大型状态横幅。处理中状态在标题附近显示，视频未就绪时继续展示明确占位状态。

### 5. 空状态

空状态只保留一句用途说明和一个聚焦链接输入框的按钮。移除巨大印章、英文口号和重复功能说明。

### 6. 页脚与弹层

页脚合并为一行版本与本地/MCP 信息。删除确认对话框、Toast 和不可逆说明保持不变，但视觉与新工作台一致。

## 响应式行为

- `>= 64rem`：左侧最近任务 + 右侧详情；详情内部视频 + 文案两列。
- `48rem–64rem`：任务栏缩窄；视频与文案仍尽量保持两列，空间不足时自动堆叠。
- `< 48rem`：解析栏换行；任务列表移动到详情上方；视频、文案、导出纵向排列。
- 所有宽度均不得产生文档级横向滚动。
- 触控目标至少 44px；保留键盘焦点、跳转链接和减少动态效果支持。

## 数据流与兼容性

现有前端轮询与 API 调用保持不变：

1. 首次加载并行请求 `/api/tasks` 与 `/api/status`。
2. 提交 `/api/tasks` 后选中新任务并立即刷新。
3. 选中任务时按需读取 metadata 与 transcript 产物。
4. 下载链接继续指向 `/api/tasks/{id}/artifacts/{artifact}`。
5. 重试、删除、Cookie 保存沿用现有端点。

HTML ID 和 `data-*` 契约尽量保留，避免无必要地修改 JavaScript。只调整任务项的装饰性序号和二级操作的容器结构。

## 错误与状态处理

- 服务断开：页头状态变为「本地服务暂时不可用」。
- 提交错误：解析栏下方显示可操作错误，不新增全页警告。
- 任务失败/部分完成：任务项及详情状态均显示；「更多」中出现重新处理。
- 文件未就绪：导出链接保持不可点击并具有 `aria-disabled`。
- 删除：继续使用原生 `dialog` 二次确认。

## 测试与验收

### 自动测试

- 集成测试验证新的结构标记、保留的无障碍标签、全部导出链接以及删除/设置入口。
- 端到端测试验证桌面端提交、紧凑解析栏、无装饰序号任务项、设置面板、Cookie 密码输入和移动端无横向溢出。
- 完整非联网测试套件必须通过。
- Ruff 必须通过。

### 浏览器验收

- 使用真实本地服务和现有已完成任务检查桌面布局。
- 在 390×844 视口检查解析栏、任务列表、视频、文案与导出顺序。
- 检查浏览器控制台无错误。
- 确认原文、语音、六个文件入口均可见，设置与删除不再抢占主视觉。

## 非目标

- 不改变抖音解析和下载策略。
- 不改变 MLX Worker、Docker 或 MCP 配置。
- 不增加搜索、筛选、分页、主题切换或新的依赖。
- 不隐藏用户要求默认可见的视频、文案和导出文件。
